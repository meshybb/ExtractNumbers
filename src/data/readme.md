# Data Preparation Module

Handles fetching, parsing, and normalization of datasets into a unified format.

## Unified Data Structure

Processed data is saved in `data/digits_data/`:

- `data/digits_data/<dataset>/sample_<id>/original.png`
- `data/digits_data/<dataset>/sample_<id>/annotations.json`

## Metadata Schema (`annotations.json`)

Standardized JSON format:

- `image_metadata`: Dimensions and index.
- `detected_numbers`: Full value, sequence bbox, and individual digit labels/bboxes.

## Key Files

- `svhn.py`, `race_numbers.py`, `handwritten.py`: Dataset-specific loaders.
- `apply_augmentations.py`: Applies Gaussian noise, blur, and geometric stretches.
- `augmentations.py`: Low-level augmentation implementations.

## Video Dataset Ground-Truth Format (e.g., DSText V2)

The downloaded video datasets (e.g., DSText V2 under `data/DSText_V2/`) are stored as raw MP4 video clips along with XML annotation files containing frame-by-frame ground-truth labels.

### Location of Ground-Truth Files
Each video (e.g., `Video_102_0_2.mp4`) in its scenario directory has an associated ground-truth XML file (e.g., `Video_102_0_2_GT.xml`).

### XML Annotation Schema
```xml
<?xml version="1.0" ?>
<Frames>
  <frame ID="1">
    <!-- Represents a detected text sequence on this frame -->
    <object ID="5752" Transcription="93" category="caption" language="alphanumeric">
      <!-- Vertices of the polygon surrounding the text crop -->
      <Point x="354" y="374"/>
      <Point x="395" y="374"/>
      <Point x="395" y="402"/>
      <Point x="354" y="402"/>
    </object>
  </frame>
</Frames>
```

- **`frame ID`**: The exact video frame index.
- **`object`**: Each individual text sequence annotated inside the frame.
  - **`Transcription`**: The actual alphanumeric text or numeric value (e.g., `"93"`, `"24"`).
  - **`Point`**: Multiple point elements outlining the precise corners/vertices of the bounding box polygon.

## Moving MNIST & RoadText Dataset Formats

We have integrated two additional video datasets to support more comprehensive sequence localization and tracking testing:

### 1. Moving MNIST
* **Location**: `data/Moving_MNIST/`
* **Annotations**: Standardized **XML annotations** (e.g., `sequence_0_GT.xml` inside `sequence_0/`) automatically generated by the pipeline.
* **Auto-Labeling Strategy**: The pipeline uses an automated OpenCV thresholding and contour detection engine to locate the moving digits frame-by-frame and export their bounding boxes with the transcription label `"digit"`.

### 2. RoadText (1K / 3K)
* **Location**: `data/RoadText/`
* **Annotations**: Master JSON annotations (`roadtext-annotation-fixed.json`) downloaded automatically from the CVC UAB dataset server.
* **Structure**: Integrates real-world street view driving scenes, containing highly representative speed limits, route numbers, address signs, and numeric sequences.

### RoadText JSON Annotation Schema
```json
{
  "1": {                  // Video sequence ID
    "12": {               // Frame ID within the sequence
      "labels": [         // List of annotated text objects on this frame
        {
          "box2d": {
            "x1": 560.12, // Bounding box top-left X coordinate
            "x2": 582.17, // Bounding box bottom-right X coordinate
            "y1": 208.76, // Bounding box top-left Y coordinate
            "y2": 218.68  // Bounding box bottom-right Y coordinate
          },
          "ocr": "50",    // The actual text/number value (e.g. speed limit "50")
          "category": "Legible",
          "id": 293
        }
      ]
    }
  }
}
```

- **`box2d`**: Standard 2D bounding box coordinate keys (`x1`, `x2`, `y1`, `y2`) outlining the crop box.
- **`ocr`**: The transcription label representing what alphanumeric sequence or digit is inside the box.

## 🐍 Python Code Examples for Reading Annotations

Here is how to programmatically parse and load the annotations for the datasets.

### 1. Parsing XML Annotations (DSText V2 & Moving MNIST)

DSText V2 and Moving MNIST use a frame-by-frame XML structure. You can load it easily using Python's standard `xml.etree.ElementTree` module:

```python
import xml.etree.ElementTree as ET

def parse_xml_annotations(xml_path):
    """
    Parses frame-by-frame XML annotation files.
    Returns a dictionary: {frame_id: [{'transcription': str, 'bbox_polygon': [(x, y), ...]}]}
    """
    tree = ET.parse(xml_path)
    root = tree.getroot()
    
    annotations = {}
    for frame in root.findall('frame'):
        frame_id = int(frame.get('ID'))
        annotations[frame_id] = []
        
        for obj in frame.findall('object'):
            transcription = obj.get('Transcription')
            # Extract polygon points defining the bounding box
            points = [(int(p.get('x')), int(p.get('y'))) for p in obj.findall('Point')]
            
            annotations[frame_id].append({
                'transcription': transcription,
                'bbox_polygon': points
            })
            
    return annotations

```

### 2. Parsing JSON Annotations (RoadText)

RoadText uses a hierarchical master JSON mapping. You can load it using Python's standard `json` module:

```python
import json

def parse_roadtext_annotations(json_path, target_sequence_id="1"):
    """
    Parses the RoadText master JSON annotation file for a target sequence.
    Returns a dictionary: {frame_id: [{'ocr': str, 'bbox': {'x1': float, 'x2': float, 'y1': float, 'y2': float}}]}
    """
    with open(json_path, 'r') as f:
        data = json.load(f)
        
    sequence_data = data.get(target_sequence_id, {})
    annotations = {}
    
    for frame_id, frame_content in sequence_data.items():
        labels = frame_content.get('labels')
        if not labels:
            continue
            
        annotations[int(frame_id)] = []
        for label in labels:
            # Bounding box coordinates: x1, x2, y1, y2
            bbox = label.get('box2d')
            ocr_text = label.get('ocr', 'unknown')
            
            annotations[int(frame_id)].append({
                'ocr': ocr_text,
                'bbox': bbox
            })
            
    return annotations

# Example usage:
# rt_data = parse_roadtext_annotations("data/RoadText/roadtext-annotation-fixed.json", target_sequence_id="1")
# print(rt_data[12]) # Show frame 12 annotations
```
